# Session 批量删除

## Purpose

用户积累了大量无用的历史 session，逐个到各 agent 的目录手工清理既繁琐又
危险。本功能让用户在列表中选择一个或多个 session，经确认后一次性批量
删除，删除过程中可以看到实时进度，删错的风险通过确认对话框控制。

## Users

需要批量清理本地 AI coding agent 历史 session 的开发者。

## Behavior

- 用户按 `x` 发起删除：普通模式下针对当前行；visual 模式下针对选中的
  连续多行。
- 发起删除后先弹出确认对话框，列出将删除的 session 数量；用户确认
  （`y` 或 `Enter`）才真正执行，按其他任何键取消且不产生任何副作用。
- 确认后删除在后台执行，界面不冻结，显示实时进度条（已完成 / 总数、
  当前正在删除的 session）。
- 删除期间不接受其他操作。
- 全部完成后列表自动刷新，状态栏给出结果摘要：成功数量与失败数量；
  单个 session 删除失败不影响其余 session 继续删除。
- 跨 agent 的混合选择（在 `all` 视图中）可以一次删除，各 session 按其
  所属 agent 的删除方式处理。

## Happy Path

1. 用户用 `V` 进入 visual 模式，用 `j`/`k`/`G`/`gg` 扩展选中范围。
2. 按 `x`，确认对话框显示将删除的 session 数。
3. 按 `y` 确认，进度条实时推进直至完成。
4. 状态栏显示「已删除 N 个 session」，列表刷新后目标 session 消失。

## Boundaries

### Included

- 单行删除与 visual mode 连续范围多选删除。
- 删除前的确认与取消。
- 后台删除、实时进度与完成后的结果摘要、列表自动刷新。
- 部分失败时继续删除其余目标并报告失败数。

### Excluded

- 删除到回收站 / 可恢复的删除（删除是不可逆的）。
- 非连续多选（如逐个标记）。
- 各 agent 具体的删除方式（属于 agent-backends）。
- 删除过程中的中途取消（当前不支持）。

## Rules

- 任何删除都必须经过确认对话框；取消路径（除 `y`/`Enter` 外的任意键）
  不得触碰任何数据。
- 进入确认时对选中目标做快照，之后列表变化不影响本次删除集合。
- 删除模式（Deleting）下忽略全部键盘输入。
- 删除完成后无论成败都刷新列表以反映真实存储状态。

## Failure Behavior

- 单个 session 删除失败：计入失败数，状态栏显示该 session 的错误，其余
  目标继续删除。
- 完成后状态栏区分「已删除 N 个」与「已删除 N 个，失败 M 个」。

## Compatibility

删除是对用户本地数据的不可逆写操作；产品上以确认对话框作为唯一的防误触
手段，不提供恢复能力。
